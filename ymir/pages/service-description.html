Service Descriptions

Service descriptions are structured service metadata which you can find stored in service.json files. Particularly important fields which you will definitely want to override are described in the paragraphs below.

The security_groups field defines which AWS security groups this service will belong to. Ymir may eventually contain helpers for building security groups, but at the moment these should probably be constructed by hand in advance.

Both the setup_list and provision_list fields both describe a list puppet files which will be invoked in standalone-mode on the remote host, in the order they are listed. For more information on the difference between setup and provisioning, please see this section of the service operations documentation.

The key_name and pem fields are critical for authenticating with your service to complete updates, etc. The key_name field refers to a named AWS key, and the pem field should be a path that points to your corresponding AWS private key file. If you don't already have a key and pem file, you can create this data using ymir: see this section of the usage-overview documentation.

Field name Type Description
org_name string the organization responsible for this service
app_name string the application this service is a part of
env_name string the environment name (suggested value: one of dev/test/prod)
service_name string the name of this service
service_description string a human-friendly description of this service
name string name of this instance (suggested value:
ymir_debug bool whether ymir is in debug mode
s3_buckets list_of_strings s3 buckets associated with this service
tags list_of_strings AWS tags which are associated with service instance
logs list_of_strings absolute paths to important logfiles on this service
elastic_ips list_of_strings elastic IP addresses associated with this service
ami string AMI used when this service is initially created
username string username used for sshing into service for provisioning, etc
instance_type string aws instance type used when creating this service
pem string a path to a keyfile on localhost that is used for sshing into the service
key_name string AWS key used when creating this instance
setup_list list_of_strings a list of instructions to carry out during service setup phase
provision_list list_of_strings a list of instructions to carry out during service provision phase
security_groups list_of_strings a list of AWS security groups this service will belong to
health_checks a hash of named checks that can be performed against this service
service_defaults free form schema-free subsection, typically used for holding application configuration data, puppet facts, etc
